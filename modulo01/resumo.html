    Resumo - Vídeo 10 - Operadores Rest/Spread

  • Sintaxe REST: const arr = [1, 2, 3, 4]; ~ const [ a, b, ...resto ] = arr;
  • Exemplo REST: Pega os 'restos' dos elementos de um array ou objeto e atribui em uma variável
  • REST: Pode ser utilizado em parâmetro de funções function soma (...params){ ...}
  • SPREAD: Propaga informação de um array ou objeto para outra estrutura de dados. Também utiliza-se ...

    Resumo - Vídeo 9 - Desestruturação

  • Sintaxe: const { nome, idade, endereco: { cidade, estado } } = usuario; onde nome e idade são propriedades do objeto usuario
  • Pode ser utilizado em funções

  • function soma(a = 2, b = 1){} -> Valores padrões em parâmetros de funções
  • Funciona em Arrow Functions

  • Sintaxe simplificada em funções anônimas

    Resumo - Vídeo 6 - Operação em Array

  • vetor.map(função(item,índice)) -> Percorre cada item do vetor e executa alguma ação.
  • vetor.reduce() -> Percorre todo o vetor e retorna apenas uma informação (ex: soma dos itens)
  • vetor.filter() -> Retorna alguma informação do vetor que for true e remove false
  • vetor.find() -> Verifica se existe alguma informação no vetor ou encontra algo no vetor

    Resumo - Vídeo 5 - Const / Let / Var

  • const -> Constante. Não pode ter o valor alterado. É mutável. Propriedades podem ser alterados. Ex: array
  • let -> Variável de escopo. Tem valor apenas em um escopo. Ex: Funções
  • var -> Variável global

    Resumo - Vídeo 4 - Classes / Constructor / Static

  • constructor( ){...} -> Executado no momento em que o objeto é instanciado.
  • super( ) -> Permite que classes herdeiras executem o construtor da classe pai.
  • static -> recebe informação e retorna algo independente do resto da classe. não é instanciável. Apenas para auxílio.

    Resumo - Vídeo 3 - Configurações Babel

  • package.json: armazena informações das dependencias instaladas. Instalado através do comando yarn init
  • Babel: Transforma features do ES7 ou ES8 para navegadores que não possuem suporte
  • yarn add @babel/cli -> Permite a utilização do babel em linha de comando
  • yarn add @babel/preset-env -> Define o ambiente de trabalho, no caso, o navegador (preset)
  • Configurar Babel: criar arquivo .babelrc na raiz do projeto
  • yarn.lock -> cache do babel
  • Script no package.JSON -> permite executar scripts a partir do terminal
  • Script dev no package.JSON faz a conversão do JS com ES6+ para navegadores sem suporte.
  • -w no fim do Script -> Atualiza automaticamente, em vez de executar o yarn dev sempre.
  • bundle.js -> arquivo que é interpretado pelo navegador com as funcionalidades do ES6+

    Resumo - Vídeo 2 - ES6+ / NodeJS / Yarn

  • Ecma Script Versão 6+: Atualizações do JavaScript
  • NodeJS: Software open-source, cross-platform, e de um runtime de JavaScript que execute código de JavaScript a nível backend e frontend.
  • Yarn: Gerenciador de pacotes do JavaScript. Auxilia na instalação de componentes JavaScript de terceiros.

    Extra: Passo a passo da configuração do Babel

  • Na raiz do projeto -> yarn init
  • Instalar dependências -> yarn add @babel/cli / yarn add @babel/preset-env / yarn add @babel/core/
  • Criar arquivo .babelrc -> para configurar o Babel
  • Conteúdo do .babelrc -> "presets" : ["@babel/preset-env"]
  • Criar arquivos html(index) e js(main)
  • No package.JSON -> Criar "scripts" : { "dev" : "babel ./main.js -o ./bundle.js -w" }
  • Executar no terminal: yarn dev para converter o main.js para bundle.js
